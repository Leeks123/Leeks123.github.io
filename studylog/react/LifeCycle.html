# 컴포넌트 생명주기

클래스 컴포넌트의 생명주기를 정리한다.

# Mounting

# constructor

React 컴포넌트의 생성자는 컴포넌트가 마운트되기 전에 호출된다.

this.state에 객체를 할당하여 지역 state를 초기화와 인스턴스에 이벤트 처리 메서드를 바인딩 과정이 진행된다.

# render()

render() 메서드는 클래스 컴포넌트에서 반드시 구현돼야하는 유일한 메서드이다.

React 엘리먼트, 배열과 Fragment, Portal, 문자열과 숫자, Boolean 또는 null 등의 값을 반환한다.

# componentDidMount()

컴포넌트가 마운트된 직후,
DOM 노드가 있어야 하는 초기화 작업,setTimeout등의 작업, 외부 데이터 호출 요청, state 업데이트 등의 작업이 이뤄진다.

# Updating

props 값이 업데이트 되었거나, setState()로 인한 업데이트 되었거나, forceUpdate()로 강제 리렌더링 될 때 진행되는 과정이다.

# shouldComponentUpdate()

props 또는 state가 새로운 값으로 갱신되어서 렌더링이 발생하기 직전에 호출된다. false를 반환하면 다음 절차가 진행되지 않는다.

성능 최적화를 위해서, 업데이트된 props, state의 값에 따른 리렌더링을 할 것인지를 결정할 수 있다.

# render()

업데이트 된 값에 따라 리렌더링 한다.

# componentDidUpdate()

render() 함수가 호출된 후 componentDidMount()와 같은 이 후 작업을 진행한다.

# Unmounting

Unmount란 컴포넌트가 제거되는 것을 의미한다.

# componentWillUnmount()

컴포넌트가 제거되기 직전에 이 함수가 호출되며, 타이머 제거, 네트워크 요청 취소, 구독 해제 등의 작업을 여기서 진행한다.
