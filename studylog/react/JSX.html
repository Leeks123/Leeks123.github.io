# JSX, 엘레먼트 렌더링, 가상 돔

# JSX

JSX는 자바스크립트를 확장한 문법으로, React 엘레먼트를 생성한다.

JSX는 Babel을 통해 트랜스 컴파일되어 자바스크립트 객체로 변환된다.

Bable
자바스크립트 코드를 변환해주는 트랜스 컴파일러.
ES6+문법등의 최신 문법을 지원하지 않는 브라우저에도 지원할 수 있도록 한다 (최신 문법을 ES5 문법으로 변환해준다.)
ES6+ 문법 뿐만 아니라, JSX, Typescript등의 트랜스파일도 지원한다.

const element = <h1 className="greeting">Hello, world!</h1>;

위와 같은 JSX 코드를 Babel은 React.createElement()로 컴파일한다.

const element = React.createElement(
  "h1",
  { className: "greeting" },
  "Hello, world!"
);

React.createElement는 다음과 같은 객체를 생성한다.

// 단순한 구조
const element = {
  type: "h1",
  props: {
    className: "greeting",
    children: "Hello, world!",
  },
};

# 엘레먼트 렌더링

ReactDOM은 React 엘레먼트와 일치하도록 DOM을 업데이트한다.

DOM에 React 엘레먼트 렌더링은 ReactDOM.render()를 이용한다

ReactDOM.render(element, document.getElementById("root"));

React 엘레먼트는 불변객체라서, 엘레먼트를 업데이트하려면 새로운 객체를 생성해야 한다.
React DOM은 해당 엘리먼트와 그 자식 엘리먼트를 이전의 엘리먼트와 비교하고 DOM을 원하는 상태로 만드는데 필요한 경우에만 DOM을 업데이트한다.

# Virtual DOM

DOM을 직접 조작해서 렌더링시키는 과정을 많은 연산이 필요한 과정이다.

Virtual DOM은 뷰에 변화가 있으면 먼저 적용되고, 최종 결과를 실제 DOM으로 전달하는 과정을 통해 연산을 줄인다. 실제로 여러 번의 리렌더링이 일어날 수 있는 과정에 대해서, 그 변화를 정리해서 한번의 렌더링으로 처리한다.

이 과정은 실제로 Virtual DOM없이도 이뤄질 수 있지만, 그 변화에 대해 파악하고 있어야 한다. VirtualDOM은 이 과정을 자동화하여 변화를 탐지하고 업데이트한다.
