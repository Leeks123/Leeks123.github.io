# 함수 호출과 This

자바에서의 this는 객체, 자기 자신을 가리킨다.
그러나 언뜻 이름이 비슷한 자바스크립트는에서의 this는 미묘하게 다르다.

this는 렉시컬 스코프를 따르지 않는다. 함수를 호출할 때 함수가 어떻게 호출되었는지에 따라 this에 바인딩할 객체가 동적으로 결정된다

# new 바인딩

# 생성자 함수의 인스턴스 생성 과정

함수 앞에 new를 붙여 생성자 호출을 하면 다음 동작들이 진행된다.

  1. 빈 객체(인스턴스) 생성
  2. 인스턴스에 프로토타입이 연결되고, this가 바인딩
  3. this를 사용하여 기술되어 있는 코드에 따라 프로퍼티, 메서드를 추가
  4. 인스턴스가 바인딩된 this를 암묵적으로 반환한다.
    명시적으로 다른 객체를 반환한다면 this가 반환되지 못한다.
function NewBinding(props) {
  this.props = props;
  this.method = function() {
    console.log("this :", this);
  };
}

const newBinding = new NewBinding("props");

newBinding.props;
// 'props'

newBinding.method();
// this : NewBinding {props: "props", method: ƒ}

# new.target

function NewBinding(props) {
  if (!new.target) {
    return new NewBinding(props);
  }
  // ...
}

생성자 함수가 new 연산자 없이 호출되는 것을 막기 위해 ES6에 도입

생성자 함수로써 호출되면 new.target 은 함수 자신을 가리킨다.
일반함수로 호출되면 undefined를 가리킨다

# 암시적 바인딩

함수 호출부에 콘텍스트 객체가 있는지를 확인하는 것이다.
다시 말해 obj.fn의 형태에서 fn함수의 내부에서의 this는 obj를 가리키게 된다.

# 메서드 호출, 함수 호출

일반 함수 호출일 경우 this는 전역 객체로 설정된다.
객체의 메서드로서의 함수를 호출한다면 this는 그 함수를 소유한 객체로 설정된다.

완전히 다른 함수 호출방식인 것 같지만, 실제로는 같다고 본다.
결국 일반 함수의 호출도 전역객체에 속한 메서드로서의 함수 호출과 같다

var a = "aaa";
let b = "bbb";

function foo() {
  console.log(this);
  console.log(this.a, this.b);
}

const obj = {
  a: 1,
  b: 2,
  foo: foo,
};

obj.foo();
// {a: 1, b: 2, foo: ƒ}
// 1 2

foo();
// Window { ... }
// aaa undefined

// window.foo();
// Window { ... }
// aaa undefined

obj.foo()의 결과로 this는 obj를 나타내고, 이 객체에 속한 a,b의 값들이 출력된다.

foo()의 결과로 this는 전역객체 window가 잡힌다. 그리고 전역 변수로 있는 a를 출력한다.

b는 왜 undefined인가?
const나 let 변수는 Window 객체의 프로퍼티로 설정되지 않는다.
실행컨텍스트 생성시 변수에 대한 정보가 저장되는 것이 다르게 지정되어 발생되는 현상인 듯 하다.

자세한 내용은 검색한 스택오버플로우의 내용을 참고하자.

주석으로 처리한 window.foo()도 같은 결과를 출력한다.
참고로 엄격모드(use strict)에서의 this는 undefined이다

# 암시적 소실

암시적으로 바인딩된 함수에서 바인딩이 소실된 경우 this는 기본 바인딩 (전역 | undefined) 이 된다.

function foo() {
  console.log(this.a);
}
function doFoo(cb) {
  cb();
}

var obj = {
  a: 2,
  foo: foo,
};

var bar = obj.foo;
var a = "엥, 전역이네";

bar(); // 엥, 전역이네
doFoo(obj.foo); // 엥, 전역이네

위의 코드에서
bar()의 경우 obj.foo의 함수 레퍼런스.
doFoo(obj.foo)의 경우 콜백으로 실행된다.

두 경우 모두 obf.foo()로 호출 된 것이 아니라 bar()와 cb()로 호출된다. 그래서 기본 바인딩이 적용된다.

정리해보면 호출된 함수의 . 앞에 붙는 객체를 this는 가리킨다.

# 명시적 바인딩. apply,call,bind

명시적으로 this를 바인딩하기 위해서는 Function.prototype에 속한 apply,call,bind를 사용한다.

function foo() {
  console.log(this.a);
}

var obj1 = {
  a: 1,
};

var obj2 = {
  a: 2,
};

foo.call(obj1); // 1
foo.apply(obj2); // 2

const boundFoo = foo.bind(obj1);
boundFoo(); // 1
  • call은 func.call(thisArg, [argsArray])로 파라미터를 배열로 받는다.
  • apply는 func.apply(thisArg, arg1,arg2,...)로 파라미터를 각각 받는다.
  • bind는 인자로 받은 객체를 바인딩한 함수를 반환한다.

# 부분적용 함수

위의 메서드로 this 바인딩 뿐만 아니라, 인수 바인딩으로도 활용할 수 있다.

function add(n1, n2) {
  return n1 + n2;
}

const addWithTwo = add.bind(null, 2);
addWithTwo(4); // 6

# 화살표 함수

위에 살펴본 this의 바인딩은 렉시컬하게(코드에 쓰인 대로(??)) 동작하지 않고 동적으로 바인딩됨을 알 수 있다.

하지만 렉시컬하게 바인딩되는 방법도 있다. 바로 ES6의 화살표 함수!!

var obj1 = {
  a: 1,
  foo() {
    return function() {
      console.log(this);
    };
  },
};

var obj2 = {
  a: 10,
  foo() {
    return () => {
      console.log(this);
    };
  },
};

obj1.foo()();
// Window { ... }

obj2.foo()();
// { a:10, foo: f }
