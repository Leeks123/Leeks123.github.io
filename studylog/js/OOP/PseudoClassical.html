# Pseudo Classical 패턴

const Foo = (function() {
  function Foo(name) {
    this.name = name;
  }
  Foo.prototype.myName = function() {
    return this.name;
  };

  return Foo;
})();

const Bar = (function() {
  const privateVar = "private variable";

  function Bar(name, label) {
    Foo.call(this, name);
    this.label = label;
  }
  Bar.prototype = Object.create(Foo.prototype);

  Bar.prototype.getPrivate = function() {
    return privateVar;
  };
  Bar.prototype.myLabel = function() {
    return this.label;
  };

  return Bar;
})();

const a = new Bar("test", "Test Label");

a.myName(); // "test"
a.myLabel(); // "Test Label"
a.getPrivate(); // "private variable"

위의 코드는 Foo를 상속받는 Bar, new Bar()로 생성하는 인스턴스 a에 관한 전형적인 프로토타입 코드이다.

  • Bar 생성자 함수에서 Foo.call(this, name)를 통해서 상위 프로토타입에 인자를 넘겼다. 클래스방식의 super의 역할을 한다.

  • Bar.prototype = Object.create(Foo.prototype)를 통해서 프로토타입을 링크한다.
    이 과정에서 Bar.prototype.constructor와 Bar 생성자 함수의 링크가 깨지게 된다.

    Bar.prototype.constructor;
// ƒ Foo(name) {
//     this.name = name;
//   }

  • a.myName()은 Foo.prototype.myName()을 호출한다.

  • a.myLabel()은 Bar.prototype.myName()을 호출한다.

    a instanceof Bar;
// true
Bar.prototype.isPrototypeOf(a);
// true
Foo.prototype.isPrototypeOf(Bar.prototype);
// true

  • privateVar는 클로져 자유번수로 동작한다. 즉 pivate 변수처럼 동작한다. 외부에서 접근 불가이다.
    위의 코드에서는 Bar.prototype.getPrivate()를 호출함으로써 변수를 반환받았다.
