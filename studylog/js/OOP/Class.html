# Class

class Foo {
  constructor(name) {
    this.name = name;
  }

  myName() {
    return this.name;
  }
}

class Bar extends Foo {
  #privateVar = "private Variable";

  constructor(name, label) {
    super(name);
    this.label = label;
  }

  myLabel() {
    return this.label;
  }
  get privateVar() {
    return this.#privateVar;
  }
  set privateVar(value) {
    this.#privateVar = value;
  }
}

# Syntactic Sugar

위의 코드와 Pseudo Classical 패턴에서의 코드를 비교해보면 위의 코드는 정말 깔끔하고 짧은 코드이다. 동작하는 역할은 같기 때문에 class를 단순히 편의 문법이라고 불리는데, 차이점은 분명히 있다.

  1. class로 만든 함수엔 특수 내부 프로퍼티인 [[FunctionKind]]:"classConstructor"가 있다.
    자바스크립트의 검증 과정을 통해 프로퍼티 여부를 확인하게 되고,
    클래스 생성자를 new와 함께 호출하지 않으면 에러가 발생한다.

  2. 클래스 메서드는 non-enumerable
    for ...in으로 객체를 순회할 시 프로퍼티만 순회할 수 있다.

  3. 클래스는 항상 엄격 모드로 실행된다.
    클래스 생성자 안 코드 전체엔 자동으로 엄격 모드가 적용된다.

# private

// ... 위의 코드에서 계속

const a = new Bar('a','label A');
a.#privateVar;
// Uncaught SyntaxError: Private field '#privateVar' must be declared in an enclosing class
a.privateVar;
// "private Variable"

#privateVar처럼 클래스 필드라는 문법을 사용하여 클래스에 프로퍼티를 추가할 수 있다.

아무런 조치 없이 클래스필드를 추가한다면 이 프로퍼티는 public 변수처럼 동작한다.

변수 명 앞에 #을 붙인다면 이 프로퍼티는 private하게 동작한다. 클래스 내부에서는 자유롭게 사용할 수 있고, 외부에서 수정,참조를 하려고 한다면 get,set을 사용하면 된다.

# static

프로퍼티나 메서드 앞에 static이라는 키워드를 붙이면 정적 프로퍼티, 메서드로 사용할 수 있다.

class Baz {
  static staticProp = "staticProp";
  static staticMethod() {
    return "staticMethod";
  }
  // ...
}

Baz.staticProp; // "staticProp"
Bax.staticMethod(); // "staticMethod"
