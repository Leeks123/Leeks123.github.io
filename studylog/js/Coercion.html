# 강제변환

# 명시적 변환

코드만 봐도 의도적으로 타입변환을 일으킨다는 사실이 명백한 타입변환.

# 숫자 타입으로 변환

  1. Number() 함수
Number("0"); // 0
Number("10.10"); // 10.1
Number(true); // 1
Number(false); // 0
  1. + 단항 연산자 활용
+"" + // 0
"0" + // 0
"1" + // 1
"string" + // NaN
true + // 1
false + // 0
null + // 0
undefined + // NaN
Symbol() + // TypeError
{} + // NaN
[] + // 0
[1, 2] + // NaN
  function() {}; // NaN

한 글자로 변환이 가능하지만,
다른 연산자와 인접하여 사용하면 다음과 같은 혼란을 야기 할 수 있으니 지양하자..

1 + - + + + - + 1; // 2

  1. 숫자 형태의 문자열 파싱
parseInt("0"); // 0
parseInt("-1"); // -1
parseFloat("10.10"); // 10.1

// 캐스팅이 아닌 결과들
parseInt(0.000008); // 0
parseInt(0.0000008); // 8 ("8e-7" => "8")
parseInt(false, 16); // 250 ("false" => "fa")
parseInt("0x16"); // 16
parseInt("103", 2); // 2

parseInt 함수는 애초에 타입 캐스팅을 위한 함수가 아니다.

파싱에 목적을 둔 함수이고 인자도 반드시 문자열이어야 한다.
인자가 비문자열이라면 암시적 강제변환이 진행된다. 그러니 피하자..

# 문자열 타입으로 변환

  1. String() 함수
String(1); // "1"
String(NaN); // "NaN"
String(Infinity); // "Infinity"
String(true); // "true"
String(false); // "false"
  1. toString()
// Object.prototype.toString
(1).toString(); // "1"
NaN.toString(); // "NaN"
Infinity.toString(); // "Infinity"
true.toString(); // "true"
false.toString(); // "false"

자바스크립트 엔진이 자동으로 객체 래퍼로 '박싱'한 후 진행한다.

# 불리언 타입으로 변환

Boolean() 함수를 활용하여 변환하는 것은 자주 쓰이지 않는다.

!! 이중 부정 연산자를 이용해 불리언으로의 강제변환을 한다.

var a = "0";
var b = [];
var c = {};

var d = "";
var e = 0;
var f = null;
var g;

console.log(a); // true
console.log(b); // true
console.log(c); // true

console.log(d); // false
console.log(e); // false
console.log(f); // false
console.log(g); // false

Falsy, Truthy

  1. Falsy (불리언으로 강제변환하면 false가 되는 값.)
  • undefined
  • null
  • false
  • +0, -0, NaN
  • ""
  1. Truthy
    1번을 제외한 나머지

# 암시적 변환

숨겨진 부수 효과로부터 발생하는 타입변환이다.

# 숫자 ~> 문자열

+ 연산자는 '숫자의 덧셈', '문자열의 접합' 두가지의 목적으로 오버로드된다.

"42" + "0"; // "420"
42 + 0; // 42 =

[1, 2] + [3, 4]; // "1,23,4"

"" + 123; // "123"

+ 연산자는 한쪽 피연산자가 문자열이거나 문자열로 변환이 가능하다면 문자열 붙이기 연산이 일어난다.
그 이외에는 숫자 덧셈을 한다.
보통 이 암시적 변환을 활용해서 ""+123과 같이 숫자를 문자열로 변환하는데 활용한다.

# 기타 ~> 문자열

NaN + '' // "NaN"
Infinity + '' // "Infinity"

true + '' // "true"
false + '' // "false"

null + '' // "null"

undefined + '' // "undefined"

(Symbol()) + '' // TypeError

({}) + '' // "[object Object]"
Math + '' // "[object Math]"
[] + '' // ""
[1,2] + '' // "1,2"
(function(){}) + '' // "function(){}"
Array + '' // "function Array() { [native code] }"

심볼의 경우 명시적 강제변환은 허용되지만, 암시적 강제변환은 금지된다.
심볼은 숫자로는 변환되지 않는다. 그러나 불리언값으로는 명시적/암시적 모두 강제변환이 가능하다.(항상 true)

# 문자열 ~> 숫자

"3.14" - 0; // 3.14

[2] - [1]; // 1

- 연산자는 숫자 뺄셈 기능이 전부이므로 "3.14"를 강제 변환한다.
배열 같은 경우 문자열로 강제변환 후 다시 숫자로 강제변환된다.
