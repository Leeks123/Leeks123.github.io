# 단축평가, 비교

# 단축 평가

&& 또는 || 연산자의 결괏값이 반드시 불리언 타입이어야 하는 것은 아니며 항상 두 피연산자 표현식 중 어느 한쪽 값으로 귀결된다
<ES5 명세>

||,&& 연산자는 첫번째 피연산자의 불리언 값을 평가한다.
피연산자가 비 불리언 타입이면, ToBoolean으로 강제변환 후 평가를 계속한다.

|| 연산자는 그 결과가 true 이면 첫 번째 피연산자 값을,
false이면 두 번째 피연산자의 값을 반환한다.

&& 연산자는 그 결과가 true 이면 두 번째 피연산자 값을,
false이면 첫 번째 피연산자의 값을 반환한다.

# 관용 코드

||,&& 연산자를 활용한 관용 코드는 다음과 같다.

  • ||
function foo(a) {
  a = a || "hello";

  console.log(a);
}

foo(); // hello
foo("hi"); // hi

디폴트 값을 할당하는데 사용되었다.

  • &&
function foo() {
  console.log(a);
}

var a = 42;
a && foo(); // 42

표현식의 가드 역할을 하는데 사용되었다.

# 옵셔널 체이닝 연산자

좌항의 피연산자가 null 또는 undefined인 경우 undefined를 반환하고, 그렇지 않으면 우항의 프로퍼티 참조를 이어감

var elem = null;
var value = elem?.value;
console.log(value); // undefined

&& 를 활용한 단축평가 대체

# null 병합 연산자

좌항의 피연산자가 null 또는 undefined인 경우 우항의 피연산자를 반환하고, 그렇지 않으면 좌항의 피연산자를 반환

var foo = null ?? "default";
console.log(foo); // 'default'

|| 를 활용한 단축평가 대체

# 느슨한/엄격한 동등 비교

동등함의 비교 시 ==는 강제변환을 허용하지만, ===는 강제변환을 허용하지 않는다.

# 추상 동등 비교 ==

비교할 두 값이 같은 타입이면 값 비교를 한다.

예외
NaN은 그 자신과도 결코 동등하지 않다.
+0와 -0는 동등하지 않다.

객체의 비교는 두 객체가 정확히 똑같은 값에 대한 레퍼런스일 경우에만 동등하다.

타입이 다를 경우는

  • 문자열과 숫자 비교

    • Type(x)가 Number이고 Type(y)가 String이면,
      x == ToNumber(y) 비교 결과를 반환한다.
    • Type(x)가 String이고 Type(y)가 Number이면,
      ToNumber(x) == y 비교 결과를 반환한다.

    즉 문자열이 숫자로 바뀐다.

  • 불리언과 다른 타입의 비교

    • Type(x)가 Boolean이면,
      ToNumber(x) == y의 비교 결과를 반환한다.
    • Type(y)가 Boolean이면,
      x == ToNumber(y)의 비교 결과를 반환한다.
    var x = true;
var y = "42";

x == y; // false

    위의 코드에서 x는 ToNumber(x)를 통해 1로 강제변환된다.
    1 == "42"의 비교에서 false가 된다.

    ToBoolean은 전혀 개입하지 않으며, truthy/falsy 여부는 전혀 무관하다.

  • null, undefined 비교

    • null과 undefined를 비교하면 서로에게 타입을 맞춰 강제변환한다.
    var a = null;
var b;

a == b; // true
a == null; // true
b == null; // true

  • 객체/함수/배열과 원시값의 비교

    • 객체에 대해 ToPrimitive로 언박싱하여 비교한다.
    var a = "abc";
var b = Object(a);

a === b; // false
a == b; // true
    • null, undefined, NaN은 해당하지 않는다.

# Corner Case

"0" == null; // false
"0" == undefined; // false
"0" == false; // true !!
"0" == NaN; // false
"0" == 0; // false
"0" == ""; // false

false == null; // false
false == undefined; // false
false == NaN; // false
false == 0; // true!!
false == ""; // true!!
false == []; // true!!
false == {}; // false

"" == null; // false
"" == undefined; // false
"" == NaN; // false
"" == 0; // true!!
"" == []; // true!!
"" == {}; // false

0 == null; // false
0 == undefined; // false
0 == NaN; // false
0 == []; // true!!
0 == {}; // false

true!!인 부분에 대해서는 주의를 하자!!
