# 프로토타입

자바스크립트는 프로토타입 기반 언어이다.

모든 자바스크립트 객체는 [[Prototype]](__proto__)라는 내부 숨김 프로퍼티를 가지고 있다. 이 프로퍼티는 다른 객체를 참조하는 레퍼런스이다.

이 프로퍼티를 이용하여 클래스 기반 상속의 기능을 비슷하게 구현할 수 있다.

# 프로토타입 체인

var objA = {
  a: 1,
};
var objB = {
  b: 2,
};
objB.__proto__ = objA;

objB.a; // 1

objB.__proto__;
// { a: 1 } == objA
objB.__proto__.__proto__;
// {constructor: ƒ, ... } == Object.prototype
objB.__proto__.__proto__.__proto__;
// null

어떤 객체에서 특정 프로퍼티 혹은 메서드를 참조하려고 한다. 그 때 객체에 해당 프로퍼티 혹은 메서드가 없다면, __proto__로 연결된 객체로 거슬러 올라가 해당 대상을 찾는다. 프로토타입 체인의 끝은 null이다.

__proto__를 사용하는 것은 권장하지 않는 방법이다.
다음의 메서드를 사용하는 것이 좋다.

  • Object.create(proto, [descriptors])
    [[Prototype]]이 proto를 참조하는 빈 객체 생성. descriptors로 프로퍼티 설명 넘길 수 있음
  • Object.getPrototypeOf(obj)
    obj의 [[Prototype]]을 반환합니다.
  • Object.setPrototypeOf(obj, proto)
    obj의 [[Prototype]]이 proto가 되도록 설정합니다.

# Property Shadowing, Method Overrides

var objA = {
  a: 1,
  prop: "objA",
  method() {
    return this.prop;
  },
};

var objB = Object.create(objA);
objB.prop = "objB";
objB.method = function() {
  return this.prop + "!!";
};

objB.method(); // "objB!!"

위의 코드에서 objA와 objB객체 모두 prop 프로퍼티와 method() 메서드를 가지고 있다.

objA와 objB는 프로토타입 체인으로 연결되어 있지만, objB는 objA의 prop과 method()를 사용할 수 없고, objB 자신의 프로퍼티, 메서드를 사용한다. 즉 objA의 prop과 method()는 가려졌다.

이 가려진 상태를 프로퍼티에 대해서는 Property Shadowing
메서드에 대해서는 메서드 오버라이딩이라는 용어를 사용한다.

# prototype과 __proto__

function Foo(name) {
  this.name = name;
}
Foo.prototype.myName = function() {
  return this.name;
};

var a = new Foo("a");
var b = new Foo("b");

a.__proto__ === Foo.prototype; // true
Foo.prototype.isPrototypeOf(a); // true

a.constructor === Foo; // true

prototype 프로퍼티는 함수 객체만 가지고 있는 프로퍼티이다.
함수 객체가 생성자로 사용될 때 이 함수를 통해 생성될 객체의 부모 역할을 하는 객체(프로토타입 객체)를 가리킨다.

prototype 객체의 constructor 프로퍼티는 자신을 생성한 객체를 가리킨다.
