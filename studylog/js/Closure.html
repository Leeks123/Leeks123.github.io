# 클로저

# 일급객체

자바스크립트의 함수는 일급객체이다.
일급객체란 보통 다른 객체들에게 적용 가능한 연산을 모두 지원하는 객체이다.

  • 리터럴로 생성 가능
  • 변수, 프로퍼티 할당 가능
  • 매개변수로 전달 가능
  • 함수의 결과로 반환 가능
  • 프로퍼티와 메서드 가질 수 있음

객체와 함수의 차이라고 한다면 함수는 호출이 가능하다는 것.

# 고차 함수 (High Order Function)

고차 함수는 함수를 인자로 받거나 반환하는 함수이다.
함수만 리턴하는 함수를 따로 일컫는 용어가 커리(Curry) 함수이다.

const discounter = (discount) => {
  return (price) => {
    return price * (1 - discount);
  };
};
// discounter = (discount) => (price) => price*(1-discount);

const tenPercentOff = discounter(0.1);
tenPercentOff(100);
// 90
discounter(0.1)(100);
// 90

# 클로저

함수와 그 함수가 선언되었을 떄의 렉시컬 환경의 조합.

자신을 포함하고 있는 외부함수보다 내부함수가 더 오래 유지되는 경우, 외부 함수 밖에서 내부함수가 호출되더라도 외부함수의 지역 변수에 접근할 수 있는데 이러한 함수가 클로저.

자신이 생성될 때의 환경(Lexical environment)을 기억하는 함수.

function outer() {
  let outerVar = "outer";
  function inner() {
    let innerVar = "inner";
    console.log(outerVar, innerVar);
  }
  return inner;
}

const closure = outer();

closure(); // outer inner

# 클로저의 활용 (모듈 패턴)

function CoolModule() {
  var something = "cool";
  var another = [1, 2, 3];

  function doSomething() {
    console.log(something);
  }
  function doAnother() {
    console.log(another);
  }

  return {
    doSomething,
    doAnother,
  };
}

var foo = CoolModule();
foo.doSomething();
foo.doAnother();

위 코드는 전형적인 모듈 패턴 구현 방식이다.
이 모듈 패턴을 사용하려면 최외곽 함수는 최소 한 번은 호출되어야 하고, 최외곽 함수는 내부 함수를 반환해야 한다.

함수 doSomething()과 doAnother()는 모듈 인스턴스의 내부 스코프에 포함하는 클로저를 가진다.
something, another 변수는 외부에서 접근할 수 없게 은닉되었다.

# 싱글톤 모듈

위의 코드는 몇 번이든 호출가능하다. 싱글톤으로 인스턴스를 생성하려면 IIFE로 바꾸면 된다.

var foo = (function CoolModule() {
  var something = "cool";
  var another = [1, 2, 3];

  function doSomething() {
    console.log(something);
  }
  function doAnother() {
    console.log(another);
  }

  return {
    doSomething,
    doAnother,
  };
})();
