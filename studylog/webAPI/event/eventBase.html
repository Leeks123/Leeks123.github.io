# 이벤트

# 이벤트 흐름


이벤트의 흐룸은 위의 그림과 같이 root 노드로부터 target 노드까지의 capturing의 과정, target 노드에서의 과정 (실제로는 별도로 처리되지 않음), 그리고 다시 root 노드로 거슬러 올라가는 bubbling의 과정이 있다.

# 캡처링

elem.addEventListener((e) => {
  console.log(e.currentTarget, "capturing phase");
}, true);

위의 코드와 같이 addEventListener의 options 파라미터에 true를 넘기면 캡처링 단계의 이벤트를 동작시킬 수 있다.

addEventListener의 options

  • once
    true이면 이벤트가 트리거 될 때 리스너가 자동으로 삭제.
  • capture
    true이면 캡처링 단계, false이면 버블링 단계에서 이벤트 다룸. 기본값은 false.
    굳이 객체로 넘길 필요 없이 true/false만 넘겨도 {capture: true/false}로 동작
  • passive
    true이면 리스너에서 지정한 함수가 preventDefault()를 호출하지 않음

이벤트의 흐름에서 주로 이용되는 것은 버블링 단계이다. 캡처링 단계는 거의 이용되지 않는다.

거의 이용되지 않는 캡처링 단계도 이용될 때가 있다.
바로 bubble 단계가 진행되지 않는 이벤트들에 대한 처리를 할 때이다.
대표적인 이벤트는 focus, blur 이벤트이다. 그외의 다른 것들은 다음의 Reference를 참고하자.

# 버블링

한 요소에 이벤트가 발생하면, 이 요소에 할당된 핸들러가 동작하고, 이어서 부모 요소의 핸들러가 동작한다. 가장 최상단의 조상 요소를 만날 때까지 이 과정이 반복되면서 요소 각각에 할당된 핸들러가 동작한다.

See the Pen Event practice by Leeks123 (@leeks123) on CodePen.


# 이벤트 객체

이벤트 객체는 이벤트 발생시 핸들러에 넘겨지는 객체로, 이벤트와 관련된 정보가 이 객체 내에 존재한다.

이벤트마다 내용이 다르지만, 공통적인 프로퍼티, 메서드는 다음과 같다.

# 프로퍼티

  1. Event.target
    실제 이벤트가 발생한 엘레먼트, 버블링이 진행되더라도 이 값은 변하지 않음
  2. Event.currentTarget
    현재 이벤트를 핸들링하고 있는 요소. this가 가리키는 엘레먼트..
  3. Event.type
    발생한 이벤트의 종류를 문자열로 반환
  4. Event.cancelable
    e.preventDefault()를 실행시켜서 기본동작을 취소시킬 수 있는지를 true/false로 반환
  5. Event.eventPhase
    이벤트 흐름의 현재 단계를 반환
    ( 0: 이벤트 없음, 1: 캡처링, 2: 타깃, 3: 버블링)

# 메서드

  1. Event.stopPropagation()
    이벤트 핸들러 함수를 통한 이벤트 처리 후 이후 버블링 단계를 진행시키지 않도록 하는 메서드
  2. Event.preventDefault()
    해당 엘레먼트가 가지는 이벤트의 기본 동작을 취소시킨다.

# 이벤트 위임

이벤트 위임은 다수의 자식 엘레먼트에 똑같은 이벤트 핸들러를 다는 대신 공통 조상 하나에만 달아서 이벤트를 처리하는 방식.

이벤트 버블링과 e.target을 활용한 방식이다.

이벤트 위임의 장점은 다음과 같다.

  • 각 하위 항목에 이벤트 핸들러를 연결하지 않고, 상위 요소에 하나의 단일 핸들러만 필요하기 때문에 메모리 사용 공간이 줄어든다.
  • 제거된 요소에서 핸들러를 해제하고 새 요소에 대해 이벤트를 바인딩할 필요가 없어진다.
